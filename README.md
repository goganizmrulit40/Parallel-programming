# Лабораторная работа №4
# Параллельное умножение матриц (CUDA)

## 1. Задание:
Модифицировать программу из л/р №1 для параллельной работы на GPU с использованием технологии CUDA. Провести серию экспериментов с:
- разными размерами матриц (200, 400, 800, 1200, 1600, 2000),
- различными конфигурациями сетки блоков (16×16, 32×32, 16×32, 32×16)*.

*Конфигурация — это размер сетки блоков (Block Size).
В CUDA каждый блок потоков имеет размерность threads_x × threads_y. Это и есть конфигурация.

|Конфигурация|	Что означает| Кол-во потоков (для 2000)
|--------|-----------|-----------|
|16×16|	Каждый блок содержит 16×16 = 256 потоков| ~3.9 млн| 
|32×32|	Каждый блок содержит 32×32 = 1024 потока|~3.9 млн| 
|16×32|	Каждый блок содержит 16×32 = 512 потоков|~3.9 млн|
|32×16	|Каждый блок содержит 32×16 = 512 потоков|~3.9 млн| 

##  2. Реализация:
### 2.1 Алгоритм умножения
Для перемножения матриц используется модифицированный алгоритм со сложностью O(n^3) - с порядком циклов i-k-j - с распределением строк матрицы A между процессами.
### 2.2 Измерение времени
Каждый эксперимент повторяется 3 раза для получения достоверных результатов.
Для точного измерения времени на GPU используются cudaEvent.
### 2.3 Метрики
|Метрика |	Формула	|Описание|
|--------|-----------|-----------------|
|Ускорение|	S = T₁ / Tₙ|	Во сколько раз быстрее с n потоками|
|Эффективность|	E = S / n × 100%|Насколько эффективно используются потоки|
|Объём задачи	|2 × n³|	Количество операций|

## 4. Настройка CUDA в Visual Studio
Скачаем и установим CUDA Toolkit 13.2 для Windows 10.
### 4.1 Настройка Visual Studio
- Свойства проекта → CUDA C/C++ → General → CUDA Toolkit Custom Dir: C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\13.2
- C/C++ → General → Additional Include Directories: добавим $(CUDA_PATH)\include
- Linker → General → Additional Library Directories: добавим $(CUDA_PATH)\lib\x64
- Linker → Input → Additional Dependencies: добавим cudart.lib

## 7. Анализ результатов
### 7.1 Масштабируемость
|Размер|	Ускорение|	Почему|
|--------|-----------|-----------------|
|200	|~64x|	Маленькая задача, GPU не полностью загружен|
|400|	~82x	|Загрузка GPU увеличивается|
|800	|~115x|	Хорошая загрузка GPU|
|1200	|~194x	|Оптимальный размер для данной конфигурации|
|1600|~237x|	Отличная загрузка GPU|
|2000	|~370x	|Максимальная загрузка GPU, 3840 ядер работают|

Вывод: Ускорение растёт с увеличением размера матрицы, достигая максимума 370x на 2000×2000.

### 7.2 Влияние конфигурации блоков
|Конфигурация	|Лучший размер|	Ускорение|	Эффективность|
|--------|-----------|-----------------|-----------------|
|16×16	|2000	|307x	|120% (сверхлинейная)|
|32×32	|2000|	370x	|36%|
|16×32	|1600|	233x|45%|
|32×16	|1200	|194x	|38%|

Вывод: Конфигурация 32×32 даёт максимальное ускорение для больших матриц. Конфигурация 16×16 показывает сверхлинейное ускорение (>100%) из-за эффектов кэша.

### 7.3 Сравнение с CPU
|Технология|	Макс. ускорение|	Время на 2000×2000|
|--------|-----------|-----------------|
|CUDA (GPU)|370x	|0.071 сек|
|OpenMP (CPU)|	5.1x	|1.391 сек|
|MPI (CPU)|	2.6x	|2.989 сек|

Вывод: CUDA на порядки быстрее CPU-технологий благодаря тысячам ядер GPU.
